//template < class type >
class Point3d
{
public:
    Point3d( type x = 0.0, type y = 0.0, type z = 0.0 ): _x( x ), _y( y ), _z( z ) {}
    type x() { return _x; }

    type _x;
    type _y;
    type _z;
}

2.c++对象模型

2.1 布局成本增加

  1. 虚函数机制（ virtual function）：支持c++执行期绑定，多态
  2. 虚基类（virtual base clas）：菱形继承，多次出现在继承体系中的base class，有一个单一而被共享的实例。
  3. 多重继承下的子类与基类之间的转换。

  4. 其他情况下，c++其实也并没有比C庞大或者迟缓。

    2.2 C++ 对象模型

    class Point
{
public:
    Point( float xval );
    virtual ~Point();
    float x() const;
    static int PointCount();
protected:
    virtual ostream&
    print( ostream &os ) const;
    float _x;
    static int _point_count;
};

    如上，在C++中有两种类成员数据（class data members）：静态非静态（static and nonstatic） 以及三种类成员函数（class member functions—static, nonstatic, and virtual），参考下面C++对象示意图， 对第一节提的问，显然是没有增加任何成本的。

非静态类成员数据被放置在每一个类实例（对象）之类，类的静态和非静态成员函数，静态数据则被放在类实例之外。 虚函数c++两步骤给予支持。

  1. 每一个class产生一堆指向虚函数指针，放在表格之中，表格成为virtual table(vtbl).
  2. 每一个类实例（对象）拥有一指向相关virtual table的指针（vptr）,每一个class 所关联的type_info object一般放在vtbl的第一个slot.

2.3 多重继承的性能探讨引子

class iostream:public istream,public ostream { ... };
class istream : virtual public ios { ... };
class ostream : virtual public ios { ... };
   ios
   / \
  ist ost
   \ /
 iostream

在C++继承体制中，“间接性”的级数因为继承的深度而增加。例如iostream需要两次间接存取才能取到ios的成员，而istream 只需要一次。当然，如果把base指针直接复制一份到子类，倒是可以直接读，空间换时间罢了。 而如果直接把基类放成员放子类中，好处：直接读写，坏处：当基类改变，所有子类全部全部得重新编译。

2.3 对象布局

c++实现动态多态： 1. 隐式转换，基类向父类转换，一般会发生截断情况。 2. virtual机制 3. 经由dynamic_cast和typeid运算符

那个一个类实例到底需要占用多少内存？ 非静态成员数据 + 加上需要字节对齐所需内存 + 为了支持virtual所需的负担
